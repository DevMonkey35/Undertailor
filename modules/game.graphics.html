Color functions

getClearColor()
Returns the game's current clear color.

Essentially, the game's background color. On each frame rendered, the screen is cleared using this color before redrawing everything else.

Defaults to black (#000000).

Returns:

    the current clear color
setClearColor(color)
Sets the game's current clear color.

See graphics.getClearColor for a brief description of clear color.

Parameters:

  • color color the color to set
getSpriteColor()
Returns the game's current sprite color.

Sprite color applies a shade on any sprites drawn in the future, should the color be anything other than white.

Defaults to white (#FFFFFF).

Returns:

    the current sprite color
setSpriteColor(color)
Sets the game's current sprite color.

See graphics.getSpriteColor for a brief description of sprite color.

Parameters:

  • color color the color to set
getShapeColor()
Returns the game's current shape color.

Shape color will be the color for any shapes drawn in the future.

Defaults to white (#FFFFFF).

Returns:

    the current shape color
setShapeColor(color)
Sets the game's current shape color.

See graphics.getShapeColor for a brief description of shape color.

Parameters:

  • color color the color to set
resetColors()
Resets all graphic colors to their default values.

This is always called at the start of every frame before drawing. Do not call this function whenever something starts to get rendered, if a specific color is needed simply set the target color, the function will check on its own whether or not it needs to change the color. Using this function to ensure the default color will slow down rendering, as it causes the game to flush any graphics to be drawn, which is slower than sending graphics all at once.

drawArc(x, y, radius, startDeg, lengthDeg[, segments=-1])
Draws an outline of an arc centered at the given point.

Parameters:

  • x number the x-coordinate of the arc's center
  • y number the y-coordinate of the arc's center
  • radius number the radius of the circle the arc outlines
  • startDeg number the degrees at which the arc will start, with 0 being at the right of the circle
  • lengthDeg number the length of the arc in degrees, starting from startDeg going counter-clockwise
  • segments number the smoothness of the arc, or -1 to automatically calculate (default -1)

drawPolygon(thickness, vargs)
Draws an outline of a polygon. The outline is closed.

Points are provided by listing out values, alternating between x and y.

The game will ignore draw calls attempting to draw polygons with less than 3 edges (6 points).

Parameters:

  • thickness number the thickness of the outline, in pixels
  • vargs ... the points of the polygon

Usage:

     -- Drawing a triangle
 graphics.drawPolygon(1.0, -- thickness
 0.0, 5.0,   -- top
 -5.0, -5.0, -- bottom left
 5.0, -5.0)  -- bottom right
drawOpenPolygon(thickness, vargs)
Draws an outline of a polygon. The outline is left open, meaning there is no line drawn between the first point and the last point.

Points are provided by listing out values, alternating between x and y.

The game will ignore draw calls attempting to draw polygons with less than 3 edges (6 points).

Parameters:

  • thickness number the thickness of the outline, in pixels
  • vargs ... the points of the polygon

Usage:

     -- Drawing a triangle
 graphics.drawOpenPolygon(1.0, -- thickness
 0.0, 5.0,   -- top
 -5.0, -5.0, -- bottom left
 5.0, -5.0)  -- bottom right
