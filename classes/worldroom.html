Class worldroom

worldrooms are container objects for worldobjects.

This class also holds the functions of the following classes:

  • destructible
  • eventlistener
  • renderable
  • processable

Methods

worldroom:getOverworld() Returns the overworldcontroller currently set with this worldroom.
worldroom:registerObject(object) Registers the provided worldobject into this worldroom.
worldroom:removeObject(object) Removes the provided worldobject from this worldroom, if the former resides inside within worldroom.
worldroom:removeObject(id) Removes the worldobject owning the provided ID from this worldroom, if the former resides within this worldroom.

Map preparation functions

While entrypoints can be added at any time during a worldroom's lifetime, it is recommended they only be added during the room's create function.
worldroom:registerEntrypoint(defPoint[, targetRoom=nil][, targetPoint=nil])
Registers a room entrypoint using the provided parameters.

Parameters:

  • defPoint string the name of the entrypoint registered by the map, in layer:point format
  • targetRoom string a path leading to the script of the target room, starting from the scripts folder (default nil)
  • targetPoint string the name of the entrypoint to spawn at in the target room (default nil)
worldroom:registerEntrypoint(defPoint[, targetRoom=nil], targetPointX, targetPointY)
Registers a room entrypoint using the provided parameters.

Parameters:

  • defPoint string the name of the entrypoint registered by the map, in layer:point format
  • targetRoom string a path leading to the script of the target room, starting from the scripts folder (default nil)
  • targetPointX number the x-coordinate of the point to spawn at in the target room
  • targetPointY number the y-coordinate of the point to spawn at in the target room

Map functions

worldroom:getCollisionLayerState(layerName)
Returns the current state of the collision layer matching the given name.

State of collision indicates whether or not the collision shapes set in the layer are active, that is, actively blocking any objects from passing through them.

Parameters:

  • layerName string the name of the collison layer to query

Returns:

    if the collision layer matching the given name is active, or false if it doesn't exist
worldroom:setCollisionLayerState(layerName, state)
Sets the current state of the collision layer matching the given name.

See wroom:getCollisionLayerState for a brief description on collision layer states.

Parameters:

  • layerName string the name of the target collision layer
  • state bool the new state of collision for the layer
worldroom:setLayerOpacity(layer, opacity)

Parameters:

worldroom:getMapDefinedShape(defShapeName)
Returns the shape data for the provided shape in this worldroom's current tilemap. The shape name must be in "layerName:shapeName" form.

Parameters:

  • defShapeName string the name of the defined shape, in layerName:shapeName form

Returns:

  1. the name of the shape type, can be "rectangle", "circle", "polygon", or "polyline"
  2. the x position of the shape's origin point
  3. the y position of the shape's origin point
  4. if a perfect circle, the radius of the circle

Or

  1. the name of the shape type, can be "rectangle", "circle", "polygon", or "polyline"
  2. the x position of the shape's origin point
  3. the y position of the shape's origin point
  4. if not a perfect circle, a table of the shape's vertices

Or

    nil, denoting the shape doesn't exist
worldroom:getMapDefinedPoint(defPointName)
Returns the coordinates for the provided point in this worldroom's current tilemap. The point name must be in "layerName:pointName" form.

Parameters:

  • defPointName string the name of the defined point, in layerName:pointName form

Returns:

  1. the x coordinate of the point
  2. the y coordinate of the point

Or

    nil, denoting the point doesn't exist
