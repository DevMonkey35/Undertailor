Class overworldcontroller

overworldcontrollers are one of the three main subsystems running under an environment.

Holds control over the Overworld layer of an environment, is the last to be rendered, and last to be processed, after the uicontroller.

This class also holds the functions of the following classes:

  • subsystem
  • processable
  • renderable
  • destructible

Methods

overworldcontroller:isCharacter(object)
Returns whether or not the provided worldobject is this overworldcontroller's character object.

It is recommended to use this instead of comparing the target object with oc:getCharacter, due to the way Lua integration works. This function simply compares the IDs of the two objects, as the IDs of worldobjects are generated per instance.

Parameters:

  • object worldobject the object to query

Returns:

    if the provided worldobject is this overworldcontroller's character object

Parameters:

  • targetEntrypoint string the name of the entrypoint to spawn the character object at in the room to be set (default nil)

Camera control functions

overworldcontroller:getCameraPosition()
Returns the current position of this overworldcontroller's camera.

The camera's origin point is in its center.

Returns:

  1. the current x position of the camera
  2. the current y position of the camera
overworldcontroller:setCameraPosition(x, y)
Sets the current position of this overworldcontroller's camera.

Parameters:

  • x number the new x position of the camera
  • y number the new y position of the camera
overworldcontroller:getCameraOffset()
Returns the current positional offset applied to this overworldcontroller's camera.

Returns:

  1. the current x offset of the camera
  2. the current y offset of the camera
overworldcontroller:setCameraOffset(x, y)
Sets the current positional offset applied to this overworldcontroller's camera.

Parameters:

  • x number the new x offset of the camera
  • y number the new y offset of the camera
overworldcontroller:getCameraZoom()
Returns the current zoom level of this overworldcontroller's camera.

The zoom level is equivalent to the area a single pixel takes on the screen, with the default value of 1 resolving to a 1:1 output on a 640x480 window.

A negative zoom level follows the same rules, but the screen will be inverted.

Returns:

    the zoom level of the camera
overworldcontroller:setCameraZoom(zoom)
Sets the current zoom level of this overworldcontroller's camera.

Parameters:

  • zoom number the new zoom level to set
overworldcontroller:isCameraFixing()
Returns whether or not this overworldcontroller's camera is constantly fixing its position.

Camera fixing ensures that the camera will never show areas of the overworld that is considered out of bounds. An area is considered out of bounds when size of the tilemap for the current room does not cover that area. By default, this value is true.

Returns:

    whether or not the camera is fixing its position
overworldcontroller:setCameraFixing(fixing)
Sets whether or not this overworldcontroller's camera is constantly fixing its position.

Parameters:

  • fixing bool if the camera should fix its position
