Class inputdata

inputdata objects are used to store information about the current state of keyboard input.

There is usually only one instance ever stored, and is constantly updated every frame.

Methods

inputdata:getPressData(key) Returns the pressdata for the provided key ID.
inputdata:isConsumed() Returns whether or not this inputdata has been consumed by another processing object for the current frame.
inputdata:consume() Consumes this inputdata for the current frame.


Methods

inputdata:getPressData(key)
Returns the pressdata for the provided key ID.

Parameters:

  • key int the key ID to query

Returns:

    the pressdata for the provided key
inputdata:isConsumed()
Returns whether or not this inputdata has been consumed by another processing object for the current frame.

Implementations should be aware of this flag; if this flag returns true, one can assume another object has processed input and wishes to take primary control. Players should naturally be controlling only one object at a time, so this flag should be respected by anything the player may be given control over.

Returns:

    whether or not this inputdata has been consumed
inputdata:consume()
Consumes this inputdata for the current frame.

See inputdata:isConsumed for details on input consumption.
