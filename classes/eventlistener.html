Class eventlistener

eventlisteners are objects that may receive certain types of data in the occurence of a scenario to generate a reaction.

Methods

eventlistener:callEvent(event) Pokes this eventlistener with an occurrence of the given event, and returns true if any response was generated.
eventlistener:handler(eventId) Registers a new handler for the specified event type ID.


Methods

eventlistener:callEvent(event)
Pokes this eventlistener with an occurrence of the given event, and returns true if any response was generated.

In their initial form, events are tables. When passing a new event to this function, it should be presented as an array table holding the ID of the event, followed by any of its parameters.

Parameters:

  • event table the event

Returns:

    if any handler responded to the event and processed it

Usage:

       local listener -- our listener
   local event = {"myEvent", "param1"}

   -- any handlers listening for "myEvent" will be triggered
   listener:callEvent(event)
eventlistener:handler(eventId)
Registers a new handler for the specified event type ID.

Handlers are functions, receiving whether or not the event being provided had previously been processed by another handler (a handler before them returned true), followed by the parameters associated with the event.

The following example continues off of the one seen in eventlistener:callEvent.

Parameters:

  • eventId string the event type ID to add a new handler for

Usage:

       local listener -- our listener

   listener:onEvent("myEvent", function(processed, p1)
     print(p1) -- will print param1
   end)
