Methods

collider:getColliderType()

Returns the body type of this collider.

This function can return 3 integers.

  • 1 - The body is a static body.
  • 2 - The body is a kinematic body.
  • 3 - The body is a dynamic body.

Returns:

    the body type id of this collider
collider:setColliderType(type)
Sets the body type of this collider.

See collider:getColliderType for the possible type IDs.

Parameters:

  • type int the ID of the body type to set

See also:

collider:applyForce(forceX, forceY[, localX=0.0][, localY=0.0])
Applies a force to this collider.

Applying a force at a point other than the object's point origin (at local 0, 0) will generate torque, if the object's rotation is not fixed (see collider:isRotationFixed).

Parameters:

  • forceX number the strength of the force to apply on the x axis
  • forceY number the strength of the force to apply on the y axis
  • localX number the location relative to this collider's origin to apply the force to on the x axis (default 0.0)
  • localY number the location relative to this collider's origin to apply the force to on the y axis (default 0.0)

collider:canCollide()
Returns whether or not this collider can collide with other colliders.

This only affects physical collision, where colliding objects stop each other from passing in their current direction. The collider will still be notified if an object collides with it, even if said object passes through it due to not having physically collided.

Returns:

    whether or not the provided collider can collide
collider:setCanCollide(canCollide)
Sets whether or not this collider can collide with other colliders.

See collider:canCollide for the effects of this property.

Parameters:

  • canCollide bool whether or not the provided collider should collide with other objects

collider:getGroupId()
Returns the group ID of this collider.

For objects that can collide, group IDs determine which objects a single object may collide with. Like with the canCollide flag, objects will still be notified if another passes through them despite having collision disabled.

Group ID rules are as follows:

  • Objects with the same group ID will not collide with each other.
  • Objects with a negative group ID collide with everything.

The default group ID for any object is -1.

Returns:

    the group ID of the provided collider
collider:setGroupId(id)
Sets the group ID of this collider.

See collider:getGroupId for a brief explanation on group IDs.

Parameters:

  • id int the new group ID of the collider
