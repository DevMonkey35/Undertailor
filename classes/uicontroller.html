Class uicontroller

uicontrollers are one of the three main subsystems running under an environment.

Holds control over the UI layer of an environment, is the first to be rendered, and is the second to be processed, before the overworldcontroller and after the scheduler.

This class also holds the functions of the following classes:

  • subsystem
  • processable
  • destructible
  • renderable

Methods

uicontroller:getUIObject(id) Returns the uiobject of the provided ID, if registered with this uicontroller.
uicontroller:registerUIObject(object, active) Registers the provided uiobject into this uicontroller.
uicontroller:removeUIObject(id) Removes the uiobject owning the specified ID, if registered with this uicontroller.
uicontroller:removeUIObject(obj) Removes the provided uiobject from this uicontroller, if registered.


Methods

uicontroller:getUIObject(id)
Returns the uiobject of the provided ID, if registered with this uicontroller.

Parameters:

  • id int the ID of the target uiobject

Returns:

    the uiobject owning the provided ID
uicontroller:registerUIObject(object, active)
Registers the provided uiobject into this uicontroller.

Parameters:

  • object uiobject the uiobject to register
  • active bool [opt=false] whether or not the object is an active object
uicontroller:removeUIObject(id)
Removes the uiobject owning the specified ID, if registered with this uicontroller.

Parameters:

  • id int the id of the target uiobject
uicontroller:removeUIObject(obj)
Removes the provided uiobject from this uicontroller, if registered.

Removal of objects differ from destruction in that the uiobject is reusable after removal, where as destruction will completely destroy the uiobject rendering it no longer usable.

Parameters:

  • obj uiobject the object to remove
