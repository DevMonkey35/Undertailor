Lua documentation for Lua libraries provided by Undertailor.

Modules

colors Colors library for creating instances of colors.
game game library for interfacing with core functionality.
game.audio audio library for interfacing with audio.
game.control control library for interfacing with user input.
game.graphics graphics library for interfacing with graphics and drawing shapes.
os An expansion of the base os library of Lua.
texts Text library for creating texts.

Classes

audio audio is the basic implementation for anything that might link to an audio track.
audiodata audiodata objects hold audio clips with parameters that may be controlled, such as volume, pitch, and pan.
audioplayable audioplayable objects give control over the playback of their contained audio track.
collider collider objects are affected by physics and can interact with each other in the Overworld subsystem.
color color objects are data representations of colors.
destructible destructible objects represent objects that may be removed from the game at some point, effectively wiping them out of the render/process loop.
environment environments are the main controller for any instance of a set of the 3 major subsystems.
eventlistener eventlisteners are objects that may receive certain types of data in the occurence of a scenario to generate a reaction.
font font objects store a set of sprites used to display language in-game.
identifiable identifiable objects have an ID by which they can be referenced by.
inputdata inputdata objects are used to store information about the current state of keyboard input.
layerable layerable objects have a notion of the concept of layers.
music music objects hold long audio tracks.
overworldcontroller overworldcontrollers are one of the three main subsystems running under an environment.
positionable positionable objects hold their position within their containers.
pressdata pressdata objects are databags for the individual state of input for specific keyboard keys, typically managed by an inputdata instance.
processable processable objects have a routine task function that gets called every frame.
renderable renderable objects have a function specifically for drawing onto the screen.
scheduler schedulers are one of the three main subsystems running under an environment.
sound sound objects hold small controllable audio tracks.
sounddata sounddata is the controlling object for instances of sounds.
sprite sprite objects hold the textures for spritesheets passed to Undertailor.
subsystem subsystems are the three subsystems of an environment, namely the scheduler, the overworldcontroller, and the uicontroller.
text text objects are databags holding a series of characters and parameters to form fully-customizable text objects.
textcomponent textcomponents are the individual segments of text in a `text object.
textstyle textstyle objects use a function to modify how single characters in rendered text objects look like.
transform transform objects are databag classes used to dictate how a renderable object should be drawn.
uicomponent uicomponents are the main processing objects inside a uicontroller subsystems.
uicontroller uicontrollers are one of the three main subsystems running under an environment.
uiobject uiobject are the main container objects for uicomponents, used in the UI layer.
worldobject worldobjects are the primary functional objects within an overworldcontroller, as well as pretty much the entire engine.
worldroom worldrooms are container objects for worldobjects.
