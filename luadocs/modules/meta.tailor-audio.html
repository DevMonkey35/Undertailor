Module meta.tailor-audio

Metatable functions for tailor-audio objects.

Both tailor-audio-sound and tailor-audio-music share these functions; some may only function for one and may be ignored by the other.

Interchangeable functions

Functions that work for both music and sound with no differences
audio:getPan()
Returns the panning value for this tailor-audio object.

Returns:

    the pan value for this piece of audio
audio:setPan(pan)
Sets the panning value for this tailor-audio object.

The panning value is how "centered" the audio is played; negative values will play audio more on the left, positives on the right, and 0 will center. 0 is the default.

Parameters:

  • pan number a number between -1.0 and 1.0 defining the panning of the audio
audio:getPitch()
Gets the currently set pitch for this tailor-audio object.

Returns:

    the pitch value for this piece of audio
audio:setPitch(pitch)
Sets the pitch value for this tailor-audio object.

Limitations of a dependency prevent the game from setting only the pitch; this function may also serve as a speed up/down function.

Parameters:

  • pitch number a number between 0.5 and 2.0 defining the pitch of the audio
audio:getPosition()
Returns the position of the currently playing audio. Only for music; sounds will always return 0.

Returns:

    the position of the currently playing music in in seconds, or 0 if this is a sound object
audio:setPosition(position)
Sets the position of the currently playing audio. Only for music; sounds will ignore this function being called.

Parameters:

  • position number the position of of the audio to set in seconds
audio:getVolume()
Returns the volume this audio will be played at.

Returns:

    the local volume of this audio
audio:setVolume(volume)
Sets the volume this audio will be played at.

This does not override global volume settings set from the game.audio library; in fact this is controlled by those values. Volume will always be calculated through master * group * local = final volume.

 local master = 1.0
 local music = 0.5
 local sound = 0.75

 local ourVolume = 1.0

 finalMusicVolume = master * music * ourVolume
 finalSoundVolume = master * sound * ourVolume

Parameters:

  • volume number a number between 0.0 and 1.0 to set the local volume of this piece of audio
audio:isLooping()
Returns whether or not this piece of audio is looping.

For music, this checks whether or not the music has a looping point. For sound, this simply checks a boolean value determining whether or not it is looping.

Returns:

    if this audio is looping
audio:setLooping(flag)
Sets whether or not this piece of audio is looping.

For both sound and music, this method simply says whether or not to replay the audio when it reaches its end.

This is the same as invoking audio:setLoopPoint(0).

Parameters:

  • flag bool whether or not to loop this audio

Non-interchangeable functions

Functions that don't work similarly for both sound and music
audio:setLoopPoint(point)
Sets the point of looping for this piece of audio.

Only useful for music, sounds will only treat the looping point value as a boolean value. Essentially, this method has no notable differences with audio:setLooping.

For music, this simply sets the reset point when playback is started again through looping.

Parameters:

  • point

Usage:

         local music;
     music:setLoopPoint(7.0); -- we're looping now
     music:play(); -- play the music

     -- when the music loops back around, it'll start at 7s in

 Setting this value to 0 or greater automagically enables
 looping.
audio:isPaused()
Returns whether or not the audio is paused.

Only works for music; sounds will always return false.

Returns:

    whether or not the audio is paused, or false if this is a sound object
audio:isPlaying()
Returns whether or not the audio is playing.

Only works for music; sounds will always return false.

Returns:

    whether or not the audio is playing, or false if this is a sound object
audio:play([volume=current volume][, pitch=current pitch][, pan=current pan])
Plays this piece of audio.

Music objects only control one piece of audio; the return value for this method has no functional use.

Sound objects manage multiple instances of the same piece of audio; this function returns the ID of the new sound that will be played.

Parameters:

  • volume number audio:setVolume (default current volume)
  • pitch number audio:setPitch (default current pitch)
  • pan number audio:setPan (default current pan)

Returns:

    nothing useful if this is a music object, or the id of the new sound to be played if this is a sound object
audio:pause([id=nil])
Pauses this piece of audio.

Music objects only control one piece of audio; the parameter is meaningless when invoked on music objects.

Sound objects manage multiple instances of the same piece of audio; if called with a specified ID, the sound of the given ID will be paused. Calling with no parameters on a sound object will pause all currently unpaused instances of the sound.

Parameters:

  • id int the id of the sound to pause if this is a sound object, or nothing to globally pause all instances of this sound or if this is a music object (default nil)
audio:stop([id=nil])
Pauses this piece of audio.

Music objects only control one piece of audio; the parameter is meaningless when invoked on music objects.

Sound objects manage multiple instances of the same piece of audio; if called with a specified ID, the sound of the given ID will be stopped. Calling with no parameters on a sound object will pause all currently playing instances of the sound.

Parameters:

  • id int the id of the sound to stop if this is a sound object, or nothing to globally stop all instances of this sound or if this is a music object (default nil)
